# Auralis Site Studio

Status: internal workflow concept

Auralis Site Studio is the internal operating model for turning client website requests into finished, maintainable websites.

This is not a public service claim by itself. It is an internal workflow for how Auralis organizes website work.

## Purpose

The purpose of Auralis Site Studio is to create a repeatable process for:

- understanding a client business
- defining website scope
- collecting required content
- building the site
- reviewing the site
- launching the site
- handing it off cleanly
- optionally supporting it through a care plan

## Core Workflow

```text
Lead or client request
→ intake
→ scope and quote
→ asset collection
→ website build
→ internal review
→ client review
→ revisions
→ launch
→ handoff
→ optional care plan
```

## Inputs

A Site Studio project should start with:

- client intake notes
- quote and scope document
- business name and contact info
- services/products list
- desired website goal
- brand assets, if available
- photos, if available
- domain/hosting information
- target launch timeline

## Outputs

A completed project should produce:

- finished website
- reviewed contact path
- confirmed ownership/access notes
- handoff checklist
- client update/care plan decision
- internal project record

## Internal Rules

- Do not start build work before scope is clear.
- Do not assume domain or hosting ownership.
- Do not promise unlimited edits.
- Do not mix care-plan work with build scope.
- Do not treat client assets as Auralis-owned.
- Do not launch until contact flow is tested.

## Role of Templates

The client intake template defines what must be known before quoting.

The quote and scope template defines what is being built and what is excluded.

The handoff checklist defines when the project can be considered complete.

The care plan documents define what happens after launch.

## Future Automation Fit

This workflow can later be supported by automation, but it should work manually first.

Potential future support:

- intake summarization
- quote draft generation
- website audit notes
- care plan reminders
- client project status summaries
- handoff packet generation

Automation should support the process, not replace client approval or owner judgment.

## Current Boundary

This document is internal process guidance only.

It does not add automation, deployment behavior, payment handling, or client portal functionality.
