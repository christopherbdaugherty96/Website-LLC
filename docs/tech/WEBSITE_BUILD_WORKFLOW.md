# Website Build Workflow

Status: operating workflow / production standard

This document defines the practical website-building workflow for Auralis Digital.

The goal is to lock down website creation first: consistent structure, strong visuals, mobile polish, and repeatable delivery.

---

## Core Rule

Do not start every website from scratch.

Use a repeatable production system:

- same intake flow
- reusable layouts
- consistent design rules
- clean mobile process
- final quality checklist

Creativity should happen inside a reliable system, not replace the system.

---

## Recommended App Stack

Install the tools that support the workflow. Do not try to master everything at once.

### Core Build Tools

**Framer** — Fast path for modern, animated, premium-looking websites.

**Figma** — Best for wireframes, layout planning, design systems, and mockups.

**Visual Studio Code** — Best free code editor for custom website work.

### Optional Code Stack (Long-Term)

**Node.js** — Needed for modern web development tools.

**GitHub Desktop** — Easy version control and backups.

**Next.js** — Premium custom-coded websites.

**Tailwind CSS** — Fast styling system for modern sites.

### Graphics and Assets

**Canva** — Quick graphics, social banners, simple brand assets, and presentations.

**Photopea** — Free browser-based Photoshop alternative.

### Content and Copy Support

**ChatGPT / AI Assistant** — Copy, ideas, proposals, SEO drafts, and refinement.

**Codex** — Code generation, layout implementation, debugging, and rapid drafts inside a repo.

Best use case:
- you provide client info + style direction + assets
- Codex builds the first draft
- you review and polish the final result

### Operations Tools

**Google Drive / Dropbox** — Client assets and backups.

**Google Sheets / Notion** — Leads, clients, tasks, and workflows.

**Google Meet / Zoom** — Client calls and reviews.

### Phone Workflow

Use your work phone for:
- photos
- videos
- client communication
- quick edits
- uploads
- maps and local visits

## Codex-Assisted Build Workflow

Use Codex as a production helper, not a replacement for judgment.

### Process

1. Collect client intake.
2. Gather photos, logo, and references.
3. Choose style direction.
4. Give Codex a clear build prompt.
5. Let Codex create the first draft.
6. Review layout, spacing, mobile, and copy.
7. Improve visuals manually.
8. Deliver the polished final version.

### Prompt Template

```text
Build a professional, mobile-first website for this business.

Business Name:
Business Type:
Location:
Main Services:
Primary Goal:
Style Direction:
Photos Available:
Logo Available:

Requirements:
- Hero
- Services
- Trust section
- Gallery
- CTA
- Footer
- Strong spacing
- Modern typography
- Subtle motion
- Mobile responsive
- Fast loading
- No fake claims

Inspect the project first, then implement cleanly.
```

### Rule

Codex accelerates production. Final quality control remains human.

## Website Focus

Ignore broader automation, passive-income ideas, and unrelated tools.

## Standard Website Structure

1. Hero section
2. Services / value section
3. Trust / proof section
4. Gallery / work section
5. Call-to-action section
6. Footer

Additional sections when needed:
- about
- testimonials
- FAQ
- pricing
- booking
- map / service area
- before-and-after gallery
- blog / updates

---

## Core Template Types

**Template A — Local Service Business**

Barbers, lawn care, detailing, contractors, cleaning, pressure washing.

**Template B — Premium Brand**

Consultants, agencies, luxury services, personal brands, high-ticket offers.

**Template C — Visual / Hospitality**

Restaurants, bars, mobile bartending, salons, and event businesses.

Reuse templates and customize them instead of rebuilding everything.

---

## Design System Rules

Define:
- two fonts maximum
- one primary color
- one accent color
- button style
- card style
- section spacing
- image treatment
- mobile text sizing

Consistency creates professionalism.

---

## Layered / Cinematic Style Rules

Good effects:
- layered hero sections
- dark overlays
- gradients
- soft shadows
- reveal animations
- hover lift
- subtle parallax

Avoid:
- too much motion
- too many colors
- inconsistent spacing
- slow effects
- unreadable text

---

## Asset Strategy

Use real photos first when possible.
Use AI/design assets only where helpful and honest.
Never fake real trust signals.

---

## Client Intake

Collect:
- business name
- business type
- services
- location

- email
- booking link
- socials
- photos / logo
- testimonials
- style preference
- competitors / references
- main goal

## Production Workflow

1. Intake review
2. Pick template
3. Wireframe
4. Write copy
5. Build visual system
6. Add assets
7. Add motion last
8. Mobile polish
9. Final QA
10. Client review

## Quality Checklist

- Clear offer in five seconds?
- CTA obvious?
- Strong mobile experience?
- Easy contact path?
- Good images?
- Consistent spacing?
- Readable typography?
- Trustworthy feel?

---

## What To Avoid

- starting from scratch every time
- overusing animations
- ignoring mobile
- endless tool switching
- polishing forever before selling

---

## Final Standard

**Good structure + good visuals + mobile polish + fast delivery + honest communication.**
