# Repo Cleanup Plan

## Goal
Make the repository easier to understand, maintain, and present to clients.

## Target Structure
- /index.html
- /projects/
- /assets/
- /docs/
- /archive/

## Recommended Moves
- Move old experiments into /archive/
- Keep active client demos in /projects/
- Keep images/fonts/media in /assets/
- Keep planning docs in /docs/
- Keep live homepage files clean at root

## Immediate Rule
Do not delete useful work. Archive first, then review.

## Benefit
A cleaner repo builds trust and makes future updates faster.
