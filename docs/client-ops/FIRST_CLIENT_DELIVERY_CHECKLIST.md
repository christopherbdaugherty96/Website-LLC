# First Client Delivery Checklist

Last updated: 2026-04-26

## Purpose

Use this when the first client says yes. The goal is to deliver cleanly, avoid scope confusion, and create a strong enough result to earn a testimonial or referral.

---

# 1. Confirm The Agreement

- [ ] Confirm client name and business name
- [ ] Confirm package / scope
- [ ] Confirm price
- [ ] Confirm payment timing
- [ ] Confirm rough timeline
- [ ] Confirm best contact method

## Minimum Written Confirmation

Before starting, get a written message or signed proposal confirming:

- what is included
- what is not included
- price
- timeline
- revision expectations

---

# 2. Intake

Collect:

- [ ] Business name
- [ ] Phone number
- [ ] Email
- [ ] Address or service area
- [ ] Hours
- [ ] Services
- [ ] Existing website or social links
- [ ] Logo / brand assets if available
- [ ] Photos if available
- [ ] Preferred customer action: call, quote, book, visit, email

---

# 3. Build Plan

Decide:

- [ ] Pages or sections needed
- [ ] Main CTA
- [ ] Trust signals to include
- [ ] Contact flow
- [ ] Mobile-first priorities
- [ ] Any demo or existing template starting point

---

# 4. First Draft

Before sending to client:

- [ ] Homepage loads
- [ ] Mobile layout is acceptable
- [ ] Contact info is correct
- [ ] CTA links work
- [ ] Placeholder text removed
- [ ] No fake testimonials or fake results
- [ ] Services are clear
- [ ] Pricing language is either approved or omitted

---

# 5. Client Review

Ask:

- [ ] Is the business information correct?
- [ ] Are the services accurate?
- [ ] Does the tone feel right?
- [ ] Are photos/logos correct?
- [ ] Is anything missing?

Keep revisions controlled and written.

---

# 6. Launch Prep

- [ ] Final content approved
- [ ] Contact path tested
- [ ] Mobile checked
- [ ] Desktop checked
- [ ] Important links tested
- [ ] SEO title/description added
- [ ] Sitemap/robots considered if separate site
- [ ] Client confirms launch

---

# 7. Handoff

Send:

- [ ] Live link
- [ ] Summary of what was built
- [ ] Any login/hosting details if relevant
- [ ] Maintenance options if offered
- [ ] How to request future edits

---

# 8. Proof Capture

After the client is happy:

- [ ] Ask for testimonial
- [ ] Ask for portfolio permission
- [ ] Capture screenshots
- [ ] Fill out `docs/CASE_STUDY_TEMPLATE.md`
- [ ] Follow `docs/CLIENT_PROOF_WORKFLOW.md`

---

# Rule

The first client is not just revenue. It is proof, process training, pricing feedback, and reputation foundation.
